Why PhishGuard AI?

Phishing attacks exploit user trust through deceptive URLs. PhishGuard AI evaluates websites using multiple analytical modules inspired by real-world phishing detection techniques.

Core System Capabilities

URL Feature Analysis

Analyzes URL length, HTTPS usage, IP-based links, subdomains, keywords, special characters, and entropy.

Risk Classification Engine

Uses a weighted scoring model to identify safe, suspicious, and phishing-prone websites.

Website Availability Check

Verifies whether the website is reachable using HTTP HEAD and GET request analysis.

Reputation Ledger

Stores analyzed URLs with trust score, hash, and timestamp for transparency and traceability.

System Workflow

  1. User submits a website URL
  2. URL format is validated
  3. Security-related URL features are extracted
  4. Risk score is calculated
  5. Website availability is verified
  6. Final trust score and verdict are generated
  7. Result is stored in reputation ledger

URL Feature Analysis

Feature Detected Value Meaning / Risk Explanation
HTTPS Enabled — HTTPS ensures encrypted communication. Absence increases phishing risk.
IP-Based URL — URLs using raw IP addresses are commonly associated with malicious sites.
URL Length — Very long URLs often hide malicious paths or misleading parameters.
Subdomain Count — Excessive subdomains (e.g. login.verify.secure.site.com) indicate spoofing.
Suspicious Keywords — Words like login, verify, bank are common in phishing URLs.
URL Entropy — High entropy indicates random or obfuscated strings used to evade detection.
